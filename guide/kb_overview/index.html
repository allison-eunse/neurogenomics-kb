    üìñ Navigating the Neuro-Omics KB¶

    Your guide to the knowledge base structure, YAML cards, and integration-critical metadata

    This page orients new readers to the structure of the knowledge base, how the YAML cards feed into the rendered docs, and where to find integration-critical metadata for gene‚Äìbrain‚Äìbehaviour foundation models.

    üéØ Scope & Evolution¶

    The KB began with adult UK Biobank‚Äìcentric gene‚Äìbrain FM alignment (genetics FM + MRI Brain FM outputs) and has expanded to include:

    • ‚úÖ 15 Model Cards: 13 foundation models (4 genetics + 5 brain + 4 multimodal/clinical) + 2 ARPA-H planning cards
    • ‚úÖ 20 Research Papers: Curated summaries with structured takeaways
    • ‚úÖ Integration Guidance: 6 YAML registries (embedding strategies, harmonization, preprocessing, alignment) + 3 narrative integration cards (ensemble integration, oncology review, multimodal FM patterns)
    • ‚úÖ Developmental/Neurodevelopmental Support: Longitudinal multimodal FMs spanning MRI/fMRI, EEG/EPhys, genetics, behavioral phenotypes
    • ‚úÖ ARPA-H BOM Alignment: Documentation for escalating from late fusion ‚Üí contrastive ‚Üí unified architectures

    üèóÔ∏è Architecture at a Glance¶

    flowchart LR
    A[External repos & datasets<br/>UKB ¬∑ HCP ¬∑ Gene FMs ¬∑ Dev cohorts] --> B[Metadata curation<br/>kb/model_cards ¬∑ kb/datasets]
    B --> C[Integration strategy & modality specs<br/>docs/integration/*]
    C --> D[Experiment configs<br/>configs/experiments/*.yaml]
    D --> E[Analysis outputs & decisions<br/>docs/decisions ¬∑ docs/generated/*]
    B --> F[Embedding & harmonization registries<br/>kb/integration_cards/*.yaml]
    F --> G[Semantic / hub alignments<br/>LLM ¬∑ VLM]
    G --> D
    • Source repos / datasets live under external_repos/ (git-ignored) and upstream storage (UKB, HCP, Cha Hospital developmental cohorts, and future ARPA-H Brain-Omics Model (BOM) data sources).
    • Metadata cards (kb/**) are the single source of truth for model/dataset specs, including adult and developmental gene‚Äìbrain‚Äìbehaviour datasets.
    • Docs pages (this site) summarize how to use those cards for integration.
    • Strategy registries (embedding/harmonization/preprocessing) connect raw exports to subject-level vectors.
    • Semantic alignment / hub registries describe how modalities (gene, brain, EEG, behaviour) are aligned to LLM and VLM spaces in ARPA-H‚Äìstyle Brain-Omics Model (BOM) projects.
    • Experiment configs log the exact recipe IDs and folds before any analysis job starts.

    Metadata you must log per run¶

    Use the CLI helpers to print the canonical recipes and copy their IDs into your run metadata:

    # Embedding (e.g., FreeSurfer PCA-512)
python scripts/manage_kb.py ops strategy smri_free_surfer_pca512_v1

# Harmonization (e.g., MURD for T1/T2)
python scripts/manage_kb.py ops harmonization murd_t1_t2

# Preprocessing stack (e.g., HCP-like rs-fMRI)
python scripts/manage_kb.py ops strategy rsfmri_swift_segments_v1

    Record:

    1. embedding_strategies.<id> for every modality in the experiment.
    2. harmonization_methods.<id> (even if it is none_baseline).
    3. rsfmri_preprocessing_pipelines.<id> whenever an rs-fMRI FM is involved.
    4. CV scheme (StratifiedGroupKFold, seed, groups) and manifest used.

    How content stays consistent¶

    • Template-first editing. Every new card starts from the templates in kb/templates/.
    • Docs ‚Üî YAML parity. If a doc cites a field (e.g., embedding recipe level), the corresponding YAML must include it.
    • Strict builds. mkdocs build --strict guards against broken navigation before publishing.

    Suggested reading order¶

    For Researchers New to Foundation Models¶

    1. KB overview (this page) ‚Äî Understand the structure
    2. Models ‚Üí Overview pages ‚Äî Browse Genetics, Brain, or Multimodal overviews
    3. Integration ‚Üí Strategy ‚Äî Learn the late-fusion-first philosophy
    4. Code walkthroughs ‚Äî Pick a model relevant to your modality
    5. Analysis recipes ‚Äî Copy-ready runbooks for CCA, prediction, etc.

    For Integration & Multimodal Work¶

    1. Integration ‚Üí Design patterns ‚Äî Late fusion ‚Üí contrastive ‚Üí unified architectures
    2. Integration ‚Üí Integration cards ‚Äî Study ensemble integration, oncology review, multimodal FM patterns
    3. Integration ‚Üí Multimodal architectures ‚Äî Deep dive into BAGEL, MoT, M3FM, Me-LLaMA, TITAN
    4. Decisions ‚Üí Integration plan (Nov 2025) ‚Äî See the escalation roadmap
    5. Integration ‚Üí Embedding policies ‚Äî Learn naming conventions for strategies

    For Adding New Models or Cards¶

    1. KB Templates ‚Äî Start from templates in kb/templates/ or generated/templates/
    2. Existing model cards ‚Äî Review kb/model_cards/*.yaml for patterns
    3. Docs ‚Üî YAML parity ‚Äî Ensure rendered docs cite fields that exist in YAML
    4. Validation ‚Äî Run python scripts/manage_kb.py validate models before committing

    Use this map whenever you add new cards or plan analyses‚Äîit keeps the documentation, YAML metadata, and experiment configs aligned.
